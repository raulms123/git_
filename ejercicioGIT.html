  1. Que es un sistema de control de versiones?

    Un sistema de control de versiones es una herramienta que gestiona cambios en el código fuente u otros archivos

  2. Cual es la diferencia entre git y github?

    Git es un sistema de control de versiones que gestiona cambios en el código fuente localmente. GitHub es una plataforma en línea que utiliza Git para alojar repositorios de código y facilitar la colaboración y la gestión de proyectos entre equipos de desarrollo.

  3. Cómo creo una rama en git?

    Usando el comando: git branch [nombre-rama]

  4. Cómo clono un repositorio?

    Usando el comando: git clone [nombre-repositorio]

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    Usando el coamando: [git pull [alias] [rama]]

  6. Cómo puedo gener un tag?

    Usando el comando: git tag

  7. Cómo hago una mezcla de ramas?

    Usando el comando: git merge [nombre-rama]

  8. Cómo puedo moverme a un commit especifico?

    Usando el comando git checkout

  9. Cómo puedo borrar una rama?

    Usando el comando: git branch -d nombre_rama

  10. Cómo puedo borrar un tag?

    Usando el comando: git tag -d [nombre_del_tag]

  11. Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Usando el comando: git git status

  12. Para que sirve el comando git remote -v?

    El comando git remote -v sirve para mostrar la lista de los repositorios remotos configurados en tu repositorio local junto con sus URLs.

  13. Cómo le digo a git cual es mi email y correo?

    git config --global user.email "tu_email@example.com" git config --global user.name "Tu Nombre"
